# New SaaS Project Checklist

1. Rename the project and branding
2. Update environment variables
3. Run the verification commands from the root `README.md`
4. Decide whether to keep billing-ready models
5. Decide whether to add real email delivery
6. Decide whether to keep or remove optional post-MVP modules such as API keys or audit logs
7. Add product-specific backend models and routes
8. Add product-specific dashboard pages
9. Update navigation, landing-page copy, and docs
10. Remove starter-kit guidance when the product is stable
