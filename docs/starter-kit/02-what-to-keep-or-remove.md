# What To Keep Or Remove

## Always keep

- Auth
- Users
- Workspaces
- Memberships
- Roles
- Settings layout
- Error handling
- Security helpers
- Docker, migrations, and verification workflow unless your team replaces them intentionally

## Optional post-MVP modules

- API keys
- Audit logs
- Password reset
- Email verification
- Real email provider
- Stripe checkout and webhooks

## Usually replace

- Landing page copy
- Branding
- Seed/demo content
- Sidebar links once product modules exist
- Billing placeholder copy if your product is not subscription-based
